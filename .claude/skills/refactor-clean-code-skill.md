# refactor-clean-code-skill

## Purpose

Force every implementation to include clean-code refactoring.

## Mandatory Questions

1. Does this already exist?
2. Can this be reused?
3. Can this be centralized?
4. Can this be shorter?
5. Can this be clearer?
6. Can this be typed?
7. Can this be tested more easily?
8. Can obsolete code be deleted?
9. Can comments be replaced by clearer names?
10. Can duplicate tests be removed?

## Refactor Targets

1. Duplicate logic
2. Duplicate literals
3. Loose strings
4. Unstructured dictionaries
5. Long argument lists
6. Unclear ownership
7. Dead compatibility code
8. Dense tests
9. Over-commented code
10. Over-engineered abstractions

## Required Output

1. Refactors required
2. Refactors completed
3. Code removed
4. Tests affected
