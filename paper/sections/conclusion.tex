\section{Conclusion}
\label{sec:conclusion}

This study holds the FedAvg AE, training protocol, and seeds fixed across B1--B4;
only the post-training threshold calibration scope varies.
On heterogeneous N-BaIoT devices, per-client calibration (B2) provides the strongest
FPR-dispersion reduction ($\Delta = 0.718$, interval $[0.647,\,0.769]$);
cluster-mean calibration (B4) recovers a meaningful but partial share
($\approx$52\% of B2's improvement) without requiring a device taxonomy, at the cost
of transmitting distributional metadata that is not private.
Family-mean calibration (B3) performs poorly: coarse device-type labels do not align
with reconstruction-error calibration structure.

When benign error distributions are near-homogeneous (CICIoT2023 under our file-level
partition, Dirichlet near-IID), no dispersion reduction is observed---B1 is sufficient.
The main tradeoff under heterogeneous conditions is that B2 reduces FPR dispersion
and improves mean Macro-F1, but worsens lower-tail P10 Macro-F1.
Per-client calibration is an FPR-equity intervention, not a uniform detection-quality improvement.

\textbf{Practitioner guidance.}
First, measure benign calibration-error heterogeneity across clients, for example
using pairwise distribution divergence or dispersion of local calibration thresholds.
Second, choose the threshold policy according to that heterogeneity and deployment
constraints: use B1 when clients are near-homogeneous or per-client state is
undesirable; use B2 when local calibration is available and false-alarm equity is the
priority; use B4 when taxonomy-free grouping is acceptable, while accounting for the
metadata leakage introduced by shared distributional fingerprints.

\textbf{Limitations.}
Regime~A uses only 9 physical devices; Regime~B uses file-level pseudo-clients, not
physical IoT devices; B4 fingerprints leak distributional metadata;
no adversarial robustness, formal privacy, or real hardware evaluation is provided;
five seeds limit statistical power (bootstrap intervals are seed-level stability summaries).
Threshold personalization is complementary to---not a replacement for---model-level personalization.

\textbf{Future work:} larger multi-vendor physical fleets,
alternative CICIoT2023 partitions (physical-device identity, attack-source, attack-type),
privacy-preserving grouping
(secure aggregation or differential privacy for fingerprint transmission),
model-level personalization comparison, adversarial robustness under poisoning
and evasion, and deployment validation under concept drift.

\noindent\textbf{Availability.}
A replication package is maintained at
\url{https://github.com/naslouby-salahe/datp}
and archived on Zenodo as release \texttt{v1.0.1} with DOI
10.5281/zenodo.20315654. It includes Hydra configuration files,
stored per-seed \texttt{metrics.json} results for Regimes A--C,
pre-computed bootstrap stability intervals, table and figure generation
scripts, README documentation, and a \texttt{COMMANDS.md} command
reference. The reported experimental results are provided as stored
per-seed result artifacts, and the main tables and statistical analyses
can be regenerated from these artifacts using the \texttt{make build-stats}
and \texttt{make build-tables} commands documented in
\texttt{COMMANDS.md}.
