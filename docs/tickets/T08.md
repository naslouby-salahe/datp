## Goal

Implement per-client conformal threshold: `τ_i = kth_smallest(E_i^cal)` where `k = ceil((n+1)*(1−alpha))`, `alpha = 1 − q` from config (default q=0.95 → alpha=0.05). Report empirical benign coverage and CV(FPR).

## Why This Exists

B2-conf provides a finite-sample coverage guarantee for FPR control under exchangeability. It strengthens the tautology defense by showing that per-client thresholds have a theoretical grounding, not just empirical behavior. Primary anchor: Lu et al. ICML 2023.

## Scope

- Implement conformal threshold computation per client
- alpha from config only (never hardcoded 0.05)
- Report empirical benign coverage on held-out test_benign data
- Report CV(FPR) under conformal thresholds
- Apply Calibration-Pending rule for insufficient calibration samples
- Report coverage ratio alongside metrics

## Out of Scope

- Claiming guaranteed attack detection (conformal guarantee is FPR-control only)
- Full conformal prediction framework (only threshold computation)
- Retraining
