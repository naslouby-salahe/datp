## Goal

Implement a score manifest verifier that validates score Parquet schema, manifests, checkpoint hashes, client IDs, and splits for all existing score cells. Emit a per-cell verification report.

## Why This Exists

Stored scores under `outputs/scores/` must be formally verified before any Group GB stored-score analyses can run. The Phase 0 preliminary reuse finding was based on metric-level agreement, but formal manifest/hash verification is required for journal claims.

## Scope

- Validate Parquet schema (column names, dtypes) matches expected spec
- Validate scoring manifest (`scoring_manifest.json`) completeness
- Validate checkpoint hash referenced in manifest exists and matches
- Validate client ID sets match the expected partition
- Validate split directories (`cal/`, `test_benign/`, `test_attack/`) are complete
- Emit a structured per-cell verification JSON report

## Out of Scope

- Recomputing metrics from scores (that is T03)
- Deciding reuse verdicts (that is T04)
- Re-training or re-scoring
