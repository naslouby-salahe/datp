## Goal

Centralize the data root into one resolver/config value so all CLI commands and sweep targets resolve to `data/processed/<slug>` canonically. Fix the diagnostic CLI default from `Path("data")` to `Path(".")`. Add conflict detection that fails loudly if both roots contain conflicting manifests.

## Why This Exists

The code currently resolves to two different processed roots depending on which command is invoked. Sweep/Makefile uses `--data-root=.` → `data/processed/<slug>`, but diagnostic CLI defaults to `--data-root=data` → `data/data/processed/<slug>`. This ambiguity must be fixed before journal experiments begin.

## Scope

- Update `src/datp/cli/diagnostic.py` default `--data-root` from `Path("data")` to `Path(".")`
- Verify all CLI entry points use the same canonical default
- Add a runtime conflict-detection function that fails if both `data/processed/<slug>/` and `data/data/processed/<slug>/` exist with conflicting manifests for the same dataset
- Update any tests that depend on the old diagnostic default

## Out of Scope

- Deleting either `data/processed/` or `data/data/processed/` directory
- Changing the `data_root()` function signature in `src/datp/data/paths.py`
- Any dataset re-processing or re-training
