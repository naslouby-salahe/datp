## Goal

If temporal feasibility is confirmed (from Edge-IIoTset or B-b data with timestamps): create chronological split, train on early window, evaluate frozen thresholds on late window, perform one-shot benign-only recalibration, and report recovery ratio.

## Why This Exists

Tests whether per-client thresholds degrade over time and whether one-shot recalibration can recover performance. This is a temporal probe, not a concept-drift solution.

## Scope

- Chronological 70/30 split per client
- Train AE on early benign data
- Calibrate thresholds on early benign calibration data
- Evaluate frozen thresholds on late data
- One-shot benign-only recalibration at boundary
- Report recovery ratio and FPR stability
- Allowed outcomes: helps, neutral, hurts/insufficient, infeasible

## Out of Scope

- Streaming drift detection
- Continuous recalibration
- Using attack labels in calibration/recalibration
