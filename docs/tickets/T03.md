## Goal

Recompute B1/B2/B3/B4 metrics from stored score Parquet files and compare against stored `metrics.json` within tolerance. Produce a `recomputed_metrics.json` artifact per cell.

## Why This Exists

Journal claims require metric reproducibility evidence. Phase 0 only confirmed aggregate-level agreement. Formal per-cell recomputation from stored scores is required to issue `VERIFIED_REUSE_SAFE`.

## Scope

- Load stored score Parquet files (cal, test_benign, test_attack) per cell
- Recompute B1, B2, B3 (Regime A), B4 thresholds and metrics using canonical functions
- Compare against stored `metrics.json` with specified tolerances
- Output `recomputed_metrics.json` with comparison results

## Out of Scope

- Re-training the encoder
- Re-scoring from checkpoints
- Modifying the threshold or metric computation logic

## Implementation Requirements

1. Function: `reproduce_cell_metrics(cell_path, config) → ReproductionResult`
2. Load score arrays per client from stored Parquets
3. Apply eligibility (n_min=100)
4. Compute thresholds using canonical `derive_threshold()` for each baseline
5. Compute metrics using canonical metric functions
6. Compare with stored `metrics.json` using tolerance table from PRE_CODING_PLAN §5.1:
   - Scalar metrics: abs ≤ 0.01
   - coverage_ratio: abs ≤ 0.001
   - Counts/ID sets: exact
   - Confusion totals: exact
7. Output structured comparison result
