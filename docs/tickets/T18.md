## Goal

Implement a model-personalization stress test (true Ditto if feasible, or FedRep-AE/FedPer-AE fallback). Evaluate whether model personalization absorbs threshold personalization.

## Why This Exists

Tests whether model-side personalization makes threshold personalization redundant. This is NOT part of the core B1–B4 ladder — it is a stress test only.

## Scope

- Decide: true Ditto or FedRep-AE/FedPer-AE fallback (document choice before training)
- Implement chosen variant
- Main µ = 0.01 unless unavailable by convergence failure
- Produce personalized checkpoints, scores, metrics
- Compute absorption ratio per the locked formula

## Out of Scope

- Making this a core baseline
- Labeling the fallback as "Ditto"
- Using a bespoke unnamed local-head baseline
- Exhaustive µ tuning

## Implementation Requirements

1. Document choice (Ditto or FedRep-AE/FedPer-AE) before implementation
2. If Ditto: proximal regularization toward global model for local updates
3. If FedRep-AE: shared encoder + local decoder heads
4. µ = 0.01 as primary; other values supplementary
5. Produce per-client personalized models and scores
6. Absorption ratio: `Δ_personalized / Δ_FedAvg` where:
   - `Δ_FedAvg = CV(FPR)[FedAvg+B1] − CV(FPR)[FedAvg+B2]`
   - `Δ_personalized = CV(FPR)[Personalized+B1] − CV(FPR)[Personalized+B2]`
7. Classification: ≥0.75 strong retention, [0.25, 0.75) partial, <0.25 near-full absorption
8. Fallback must be a recognized personalization family — never bespoke

## Refactoring Requirements

- If FedRep style: may need to separate encoder/decoder in model architecture
- Reuse FL infrastructure where possible
- Clear separation from core B1–B4 paths
