## Implementation Requirements

1. Document choice (Ditto or FedRep-AE/FedPer-AE) before implementation
2. If Ditto: proximal regularization toward global model for local updates
3. If FedRep-AE: shared encoder + local decoder heads
4. µ = 0.01 as primary; other values supplementary
5. Produce per-client personalized models and scores
6. Absorption ratio: `Δ_personalized / Δ_FedAvg` where:
   - `Δ_FedAvg = CV(FPR)[FedAvg+B1] − CV(FPR)[FedAvg+B2]`
   - `Δ_personalized = CV(FPR)[Personalized+B1] − CV(FPR)[Personalized+B2]`
7. Classification: ≥0.75 strong retention, [0.25, 0.75) partial, <0.25 near-full absorption
8. Fallback must be a recognized personalization family — never bespoke

## Resolution Record

**Decoder persistence blocker — now resolved (2026-05-25):**

The prior `NotImplementedError` guard in `run_fedrep_training()` has been removed. The following was implemented:

- `DatpFedRepClient.__init__()` now takes `decoder_ckpt_dir: Path`. After each `fit()` call (both training phases complete), the client saves `model.decoder.state_dict()` atomically to `decoder_ckpt_dir / {cid} / decoder.pt`.
- `run_fedrep_training()` calls `run_fl_simulation(..., score_after=False)` to train and checkpoint the aggregated encoder without scoring. After training, it loads the aggregated encoder state once, then for each client loads that client's saved decoder, assembles the full personalized model, and calls `score_fedrep_clients()` to produce per-client score parquets and the scoring manifest.
- `score_fedrep_clients()` added to `src/datp/training/scoring.py`: takes `client_models: dict[str, Autoencoder]` and produces the same manifest/sentinel structure as `score_clients()`.
- `SimClientConfig` dataclass added to `src/datp/training/simulation.py` to group `client_cls`, `client_extra_kwargs`, `encoder_only`, `score_after`; reduces `run_fl_simulation` parameter count back within the quality limit.
- `DECODER_CHECKPOINT = "decoder.pt"` constant added to `src/datp/artifacts/constants.py`.
- Tests updated in `tests/unit/training/protocols/test_fedrep.py`: decoder persistence (3 cases), `score_fedrep_clients` (3 cases), `run_fedrep_training` end-to-end with mocked FL (1 case). All 41 tests in the affected suite pass.

**What is implemented:**
- `DatpFedRepClient` in `src/datp/training/protocols/fedrep.py`: encoder aggregated via FedAvg, per-client decoder saved after each `fit()`.
- `get_parameters()` returns only encoder parameters — decoder is local-only.
- `fit()` runs Phase 1 (decoder only, encoder frozen) then Phase 2 (full model), then saves decoder checkpoint.
- Naming guard: class and module explicitly state "NOT Ditto"; test enforces this.
- `run_fedrep_training()`: trains with `score_after=False`, loads per-client decoders, scores all clients with personalized models.
- `AbsorptionClass` enum and `classify_absorption()` in `src/datp/core/enums.py`.
- `ABSORPTION_STRONG_RETENTION_THRESHOLD = 0.75`, `ABSORPTION_PARTIAL_THRESHOLD = 0.25`.

**Checkpoint paths:**
- Aggregated encoder: `base_dir/fedrep/seed_{seed}/model.pt`
- Per-client decoder: `base_dir/fedrep/seed_{seed}/{client_id}/decoder.pt`
- FedRep-AE scores: `base_dir/fedrep/seed_{seed}/scores/{stage}/{client_id}.parquet`
