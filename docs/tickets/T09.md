## Goal

Implement the B-FedStatsBenign benign-only federated summary-statistics comparator per the protocol lock. Produce comparator table with between_ratio diagnostic.

## Why This Exists

Reviewers may demand a Laridi-style comparator. B-FedStatsBenign is the DATP-compatible benign-only adaptation. It tests whether a benign-only federated summary-statistics threshold can match or improve on B2.

## Scope

- Client summaries: count, mean, variance of benign calibration errors
- Sample-count-weighted global mean
- Full pooled variance with between-client mean-shift term
- Threshold grid: `τ(k) = μ_global + k · σ_global`, `k ∈ {0.00, 0.01, ..., 5.00}`
- Primary operating point: `k* = argmin_k |exceedance(k) − 0.05|`, tie-break: larger k
- Outputs: achieved exceedance, absolute deviation from 5%, between/within variance terms, between_ratio, CV(FPR), Macro-F1, coverage
- No attack labels in threshold computation
- No post-hoc tuning

## Out of Scope

- `B-LaridiFaithful` (out of DATP scope — requires anomaly-labeled calibration)
- Fixed-k variants in main paper (supplement only)
- Claiming attack-labeled superiority

## Implementation Requirements

1. Per-client summaries: `(n_i, μ_i, σ²_i)` from benign calibration errors
2. Weighted global mean: `μ_global = Σ(n_i × μ_i) / Σ(n_i)`
3. Pooled variance: `σ²_global = within_var + between_var`
   - `within_var = Σ(n_i × σ²_i) / Σ(n_i)`
   - `between_var = Σ(n_i × (μ_i − μ_global)²) / Σ(n_i)`
4. `between_ratio = between_var / σ²_global`
5. Threshold grid: `τ(k) = μ_global + k × sqrt(σ²_global)`, k from 0.00 to 5.00 step 0.01
6. Select `k*`: minimize |exceedance - 0.05|, tie-break larger k
7. Evaluate metrics at k*
8. No attack labels used in any computation
